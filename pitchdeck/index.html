LI.FI — CROSS-CHAIN ONBOARDING
ETH / ARB / BASE
→
LI.FI SDK
→
HyperEVM + USDC
  • @lifi/sdk npm package
  • User connects wallet on source chain
  • SDK finds optimal route to HyperEVM
  • Handles token swaps if needed
  • Single transaction execution
  • User lands with USDC ready
import { getQuote, executeRoute } 
  from '@lifi/sdk'

const quote = await getQuote({
  fromChain: 'ETH',
  toChain: 999, // HyperEVM
  fromToken: 'USDC',
  toToken: 'USDC',
  fromAmount: amount,
  fromAddress: userAddress
})

await executeRoute(quote)

Valantis Integration
N E M E S I S

Protocol treasury earns yield on staked HYPE. Your escrow stays safe in USDC.

VALANTIS — PROTOCOL TREASURY YIELD
Fees → HYPE
→
Valantis stHYPE
→
~8% APY
  • Protocol treasury accumulates HYPE from fees
  • HYPE staked to Valantis stHYPE contract
  • Liquid staking — remains liquid
  • Yield accrues to protocol treasury

Separation of Concerns

User funds (duel escrow) remain in USDC at all times. Only protocol-owned funds stake to Valantis. Users have defined risk without HYPE price exposure.

SALT — NON-CUSTODIAL AUTOMATION

Robo Guardians

Automated co-signers on user's AWS. Only co-sign txs passing policy checks.

Robo Managers

Third-party bots executing without custody. Nemesis IS a Robo Manager.

POLICY TYPES
  • Allowed Recipients: Whitelist (Pear, Valantis)
  • Denied Recipients: Blacklist
  • Transaction Limits: Max per tx/period
  • Nominated Approvers: Human sign-off
import { SaltClient } from 'salt-sdk'

const client = new SaltClient()
await client.authenticate(siweMessage)

const org = await client.createOrg()
await org.setPolicy({
  allowedRecipients: [PEAR, VALANTIS],
  txLimit: parseEther('100')
})
